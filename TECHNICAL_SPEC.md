# ТЕХНИЧЕСКОЕ ЗАДАНИЕ: Главная страница → Первый экран «Приветствие»

## 1. Назначение блока
Первый экран — это **входной системный экран**, задающий стиль всего сайта.  
Он должен выглядеть как интерфейс, а не как маркетинговый баннер.

## 2. Общая композиция
Экран состоит из четырёх основных зон:

1. **Фоновый слой**  
   - Тёмный, глубокий фон.  
   - Допускается лёгкая текстура или сетка.  
   - Никаких изображений, фотографий, градиентов.

2. **Левый UI‑блок**  
   - Вертикальные линии (2–4 шт).  
   - Малые точки‑индикаторы (3–6 шт).  
   - Расположены строго по вертикальной оси слева.  
   - Никакой декоративности — только системный стиль.

3. **Центральное ядро**  
   - Заголовок: «Добро пожаловать».  
   - Подзаголовок: «Это вход в систему».  
   - Разделительная линия между ними.  
   - Кнопка/элемент действия: «Продолжить».  
   - Центрирование по вертикали и горизонтали.  
   - Стиль — строгий, интерфейсный, без украшений.

4. **Правый UI‑блок**  
   - Вертикальные линии (1–3 шт).  
   - Точки‑индикаторы (2–4 шт).  
   - Симметричное, но не зеркальное расположение относительно левого блока.

5. **Нижняя панель**  
   - Линия или ряд точек (3–6 шт).  
   - Выполняет роль «системного статуса».

---

## 3. Функциональные требования
- Экран занимает **100% высоты окна**.  
- Контент не прокручивается — это отдельный экран.  
- Переход к следующему экрану по кнопке «Продолжить» или по скроллу.  
- Анимации допускаются, но должны быть минимальными:  
  - лёгкое появление,  
  - плавные линии,  
  - микродвижение точек.

## 4. Стиль и визуальный язык
- Цветовая палитра: чёрный, графитовый, золотой акцент (минимально).  
- Линии — тонкие, ровные, без свечения.  
- Точки — маленькие, строгие, без бликов.  
- Шрифт — строгий, геометричный, без округлостей.  
- Никаких картинок, иконок, иллюстраций.

## 5. Адаптивность
- На мобильных:  
  - Левый и правый UI‑блоки упрощаются (меньше линий и точек).  
  - Центральное ядро остаётся неизменным.  
- На широких экранах UI‑блоки расширяются.

## 6. Ограничения
- Никаких декоративных элементов.  
- Никаких текстов, кроме указанных.  
- Никаких внешних библиотек.  
- Никаких изображений.  
- Никаких анимаций, которые отвлекают.

## 7. Результат
Разработчик должен предоставить:

1. Готовый первый экран, полностью соответствующий структуре.  
2. Чистую, модульную верстку.  
3. Строгий интерфейсный стиль.  
4. Полную совместимость с текущей архитектурой сайта.
